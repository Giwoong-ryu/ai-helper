슬래시 커맨드 17개 (LEVEL 1)

자동화

슬래시(/) 명령어로 반복 작업을 한 번에 처리

/analyze코드/시스템 분석

코드베이스 전체를 분석하여 아키텍처 패턴, 의존성, 잠재적 문제점 탐지

/build빌드/컴파일 실행

프로젝트 자동 빌드, 에러 자동 감지 및 수정 제안, npm/cargo build 자동 감지

/cleanup코드 정리/포맷팅

불필요한 코드 제거, 자동 포맷팅, import 최적화, 주석 정리, Prettier/ESLint 실행

/design아키텍처/시스템 설계

새 기능 또는 시스템 설계. 아키텍처 다이어그램 생성, 컴포넌트 구조 제안

/document문서 자동 생성

README, API 문서, JSDoc/Docstrings 자동 생성, 사용 예제 포함

/estimate작업 추정/계획

작업 시간 추정, 태스크 분해, 우선순위 제안, 스토리 포인트 및 리스크 분석

/explain코드/개념 설명

복잡한 코드 로직을 쉬운 언어로 설명, 알고리즘 및 디자인 패턴 분석

/save-work작업 내용 자동 저장

하루 작업 내용 자동 정리 및 요약, 날짜별 히스토리 관리

/implement기능 구현

요구사항 기반 코드 작성, 베스트 프랙티스 적용, 타입 안전성 보장, 에러 핸들링 포함

/improve코드 개선 제안

성능 최적화, 가독성 향상, 보안 강화, SOLID 원칙 적용

/index프로젝트 인덱싱

코드베이스 전체 구조 파악, 파일 관계 맵핑, 주요 진입점 파악

/load컨텍스트 로드

특정 파일/폴더를 작업 컨텍스트에 로드, 관련 파일 자동 감지

/spawn파일/컴포넌트 생성

새 파일/컴포넌트 스캐폴딩, 보일러플레이트 생성, 템플릿 기반 코드 생성

/task복잡한 작업 실행

복잡한 멀티스텝 작업 자동 수행, Agent 시스템과 연동

/test테스트 생성/실행

단위/통합 테스트 자동 생성, Jest/Pytest 자동 감지, 테스트 커버리지 개선 제안

/troubleshoot문제 진단/해결

버그 자동 진단, 로그 분석, 에러 트레이스 추적, 즉시 수정 적용

/workflow워크플로우 자동화

CI/CD 파이프라인 생성, GitHub Actions 설정, 배포 자동화, 전체 개발 워크플로우

Progressive Disclosure 방식

필요한 정보만 단계적으로 로드하여 토큰을 효율적으로 사용

  • 1. 스킬 이름만 로드 (최소 토큰)
  • 2. 필요 시 상세 설명 로드
  • 3. 실행 시에만 전체 컨텍스트 로드

설치 방법

.claude/skills/ 폴더에 스킬 파일 추가

MCP (Model Context Protocol)

연동

파일, 데이터베이스, API 등 외부 데이터를 실시간으로 직접 읽고 쓸 수 있게 연결

MCP로 연결 가능한 것

  • 파일 시스템

    로컬 파일 직접 읽기/쓰기

  • 데이터베이스

    PostgreSQL, MySQL, MongoDB 등

  • Git 저장소

    GitHub, GitLab 직접 연동

  • 외부 API

    REST API, GraphQL 연동

설치 방법

1. MCP 서버 설치

npm install @modelcontextprotocol/server-filesystem

2. Claude 설정 파일에 추가

claude_desktop_config.json 수정

3. Claude 재시작

이후 파일 복사붙여넣기 없이 직접 접근 가능
